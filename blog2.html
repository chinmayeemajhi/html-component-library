var and function hoisting in javaScript

December 11, 2021

Hoisting is a very popular concept in JavaScript, due to which we can access variables and funtions even before declaring them.

But What is Hoisting?

According to the definition of hoisting, " It is the JavaScriptâ€™s behaviour of moving declarations (functions and variable) to the top of their current scope". So is it physically moved to the top of your code? The answer is No. JavaScript Engine doesn't move your code anywhere. It stays the way you typed it. Then, how hoisting is possible? The answer lies in the "Execution Context".

What is Execution Context?

Execution Context is an environment where JavaScript code runs. The code excution happens in two phases: Memory Phase, then Code Execution Phase.

In memory phase, memory is allocated to var and functions even before executing a single line of code. It is also called Variable Environment. JavaScript Engine scans the code and stores vars and functions as key value pair in the variable environment. In key the function and var name is stored. But what is there in the value? Remember javascript only hoists the declarations not initializations, The actual value assigned is stored later in the code execution phase. So for vars a special keyword "undefined" is stored and for functions in the value the whole function is stored.

While code execution takes place and we try to access a var or function before declarations, it does not throw any error. Because the vars and functions have already been stored in the scope during the memory phase. And this is why it appears as if all the declarations are moved to the top.
